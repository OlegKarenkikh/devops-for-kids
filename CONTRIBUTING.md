# 🤝 Как внести вклад в курс

Спасибо за интерес к проекту! Этот курс создан для всех — от детей до взрослых, кто хочет понять DevOps с нуля.

## Как помочь

### 🐛 Нашёл ошибку
Открой [Issue](https://github.com/OlegKarenkikh/devops-for-kids/issues) и опиши:
- В каком уроке ошибка (ссылка на файл)
- Что написано сейчас
- Что должно быть правильно

### ✍️ Хочешь улучшить объяснение
1. Fork репозитория
2. Создай ветку: `git checkout -b improve/lesson-name`
3. Внеси изменения
4. Открой Pull Request с описанием что и зачем изменил

### 💡 Идея нового урока
Открой Issue с тегом `enhancement` и опиши:
- Тема урока
- Для кого (уровень: новичок / продвинутый)
- Почему это важно для курса

## Принципы курса

При создании контента придерживайся этих правил:

| Принцип | Как это выглядит |
|---------|-----------------|
| **Простой язык** | Объясняй как 12-летнему ребёнку без опыта |
| **Аналогии** | Каждый новый концепт — через понятную метафору |
| **Практика сразу** | После каждого объяснения — команда которую можно запустить |
| **Ошибки как урок** | Показывай типичные ошибки и как их читать |
| **Без магии** | Объясняй что происходит под капотом, не просто «вводи это» |

## Структура урока (шаблон)

```markdown
# 🎯 Урок N — Название

> Для кого этот урок (1-2 строки)

## Аналогия — что это такое простыми словами

<img...> + подпись

Объяснение через знакомый пример из жизни.

## Практика — запусти прямо сейчас

```bash
# Комментарий что делает команда
команда
# Ожидаемый результат
```
