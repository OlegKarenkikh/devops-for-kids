# 🎨 Иллюстрация: module6-env-docker

## Назначение
Показать как переменные из .env файла попадают внутрь Docker контейнера.

## Схема
```
Файл .env (на хосте)          Контейнер
┌──────────────────┐          ┌──────────────────┐
│ DB_PASSWORD=xxx  │  ──────► │ $DB_PASSWORD=xxx │
│ API_KEY=yyy      │  ──────► │ $API_KEY=yyy     │
└──────────────────┘          └──────────────────┘
       ↑                              ↑
  --env-file .env              os.environ["DB_PASSWORD"]
  или env_file: .env (Compose)
```

### Ключевые надписи:
- Иконка замка на .env: "не коммитим в git"
- Стрелка с надписью "docker run --env-file" или "env_file:"
- Внутри контейнера: переменные доступны через os.environ

## Стиль
Две коробки: хост и контейнер. Стрелки передачи. Тёмный фон.
