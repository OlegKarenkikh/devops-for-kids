# 🎨 Иллюстрация: module2-ssh-server-vs-github

## Назначение
Устранить путаницу между SSH для сервера (Модуль 1) и SSH для GitHub (Модуль 2).

## Схема — два сценария рядом

### Левая половина: SSH → Сервер (Модуль 1)
```
[Твой компьютер] ──SSH──> [Сервер 192.168.1.1]
Ключ кладём в:
~/.ssh/authorized_keys НА сервере
Проверка: ssh user@server-ip
```

### Правая половина: SSH → GitHub (Модуль 2)
```
[Твой компьютер] ──SSH──> [github.com]
Ключ кладём в:
github.com → Settings → SSH Keys (в браузере)
Проверка: ssh -T git@github.com
```

### Общий элемент посередине:
```
[~/.ssh/id_ed25519.pub]  ← один и тот же публичный ключ
        ↙                           ↘
на сервер                      на GitHub
```

### Надпись снизу:
"Инструмент один — сценарии разные"

## Стиль
Разделённая схема. Левая и правая части разного цвета (синий/зелёный).
