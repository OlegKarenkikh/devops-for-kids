# 🌿 Модуль 2 — Git и GitHub (Уроки 6–8)

> **Цель:** научиться сохранять историю изменений, работать с ветками и делиться кодом на GitHub.

---

## 📖 Урок 6 — Git: сохраняем работу

### 🧠 Объяснение

Помнишь, в видеоиграх есть «сохранения»? Ты нажал сохранить — и если проиграешь, вернёшься к этому месту. **Git** — это точно такая же система сохранений, только для кода. Каждое сохранение называется **коммит**.

### 💻 Установка и настройка

```bash
sudo apt install git        # Установка на Ubuntu
git --version               # Проверить версию

# Настройка (один раз):
git config --global user.name "Твоё Имя"
git config --global user.email "твой@email.com"
```

### 💻 Основные команды

```bash
git init                    # Начать Git-проект
git status                  # Что изменилось?
git add файл.txt            # Добавить файл к сохранению
git add .                   # Добавить ВСЕ изменения
git commit -m "Описание"    # Сделать сохранение
git log --oneline           # История сохранений
git diff                    # Что изменилось с последнего коммита?
```

### 📊 Схема работы Git

```
Рабочая папка  →  git add  →  Staging  →  git commit  →  История
(изменяешь)                  (готово к                  (сохранено!)
                              сохранению)
```

### 🧪 Задание

```bash
mkdir мой-сайт && cd мой-сайт
git init
echo "# Мой первый сайт" > README.md
git add .
git commit -m "🎉 Первый коммит"
git log --oneline
```

## 📖 Урок 7 — Ветки и слияние

### 🧠 Объяснение

Ветка — это как параллельный мир твоего кода. Хочешь попробовать идею — создай ветку. Получилось — слей с основной. Не получилось — просто удали ветку.

### 💻 Команды

```bash
git branch                  # Список веток
git branch моя-идея         # Создать новую ветку
git checkout моя-идея       # Переключиться на ветку
git checkout -b новая-ветка # Создать и сразу переключиться
git merge моя-идея          # Влить ветку в текущую
git branch -d моя-идея      # Удалить ветку
```

### 🧪 Задание

```bash
git checkout -b добавляю-список
echo "- пункт 1" >> README.md
echo "- пункт 2" >> README.md
git add . && git commit -m "Добавил список"
git checkout main
git merge добавляю-список
git log --oneline
```

## 📖 Урок 8 — GitHub: делимся с миром

### 🧠 Объяснение

Git — сохранения у тебя на компьютере. **GitHub** — облако, куда ты отправляешь сохранения, чтобы они не потерялись и другие могли работать с тобой вместе.

### 💻 Команды

```bash
# Привязать локальный проект к GitHub:
git remote add origin https://github.com/ТОЁ_ИМЯ/РЕПО.git

# Отправить изменения в облако:
git push origin main
git push -u origin main     # -u запомнит ветку (один раз)

# Получить изменения из облака:
git pull

# Скачать чужой проект:
git clone https://github.com/имя/репо.git
```

### 🧪 Задание

1. Зарегистрируйся на [github.com](https://github.com)
2. Создай новый репозиторий (New → Repository)
3. Выполни:

```bash
git remote add origin https://github.com/ТОЁ_ИМЯ/мой-сайт.git
git branch -M main
git push -u origin main
```

## 📋 Шпаргалка Модуля 2

| Команда | Что делает |
|---------|-----------|
| `git init` | Начать репозиторий |
| `git status` | Статус изменений |
| `git add .` | Добавить всё |
| `git commit -m "текст"` | Сохранить |
| `git log --oneline` | История |
| `git branch имя` | Создать ветку |
| `git checkout имя` | Переключиться |
| `git merge имя` | Слить ветку |
| `git push` | Отправить в облако |
| `git pull` | Получить из облака |
| `git clone URL` | Скачать проект |
