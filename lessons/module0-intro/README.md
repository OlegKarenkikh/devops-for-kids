## Глава 1 — Большая картина: как работает любое приложение

<div align="center">
<img src="https://raw.githubusercontent.com/OlegKarenkikh/devops-for-kids/main/images/intro_three_layers.png" alt="Три слоя любого приложения" width="90%"/>
<br/><em>Google, YouTube, ВКонтакте — все работают по одной схеме из трёх слоёв</em>
</div>

### Метафора: ресторан

Представь ресторан:
- **Официант** принимает заказ у гостя → это **Frontend** (сайт, мобильное приложение)
- **Повар** готовит блюдо по рецепту → это **Backend** (наш Flask API, логика)
- **Холодильник и кладовая** хранят продукты → это **База данных** (PostgreSQL, SQLite)

Когда ты открываешь YouTube и нажимаешь «Воспроизвести» — за 0.1 секунды происходит:
1. Браузер отправляет HTTP-запрос на сервер YouTube
2. Backend проверяет твой аккаунт в базе данных
3. База возвращает ссылку на видео-файл
4. Backend отдаёт ответ браузеру
5. Браузер воспроизводит видео

**Ты создашь точно такую же систему** — маленькую, но настоящую.

## Глава 2 — Путь кода: от идеи до пользователя

<div align="center">
<img src="https://raw.githubusercontent.com/OlegKarenkikh/devops-for-kids/main/images/intro_devops_pipeline.png" alt="DevOps Pipeline" width="90%"/>
<br/><em>Это называется DevOps Pipeline — автоматический конвейер от кода до продукта</em>
</div>

### Метафора: автозавод

На заводе Lada каждая машина проходит конвейер — сварка, покраска, сборка, контроль качества, отгрузка. Никто не собирает машину вручную с нуля каждый раз.

Код в больших компаниях работает **так же**: программист пишет изменение, нажимает `git push` — и автоматически запускается конвейер (pipeline):

| Этап | Инструмент | Что делает |
|------|-----------|-----------|
| **Написал код** | Редактор + Терминал | Создаёшь или меняешь файлы |
| **Сохранил историю** | Git | Фиксирует кто, что и когда изменил |
| **Упаковал** | Docker | Собрал контейнер с кодом и зависимостями |
| **Проверил** | GitHub Actions | Запустил тесты автоматически |
| **Доставил** | Kubernetes | Развернул новую версию без простоя |
| **Наблюдает** | Prometheus + Grafana | Мониторит что всё работает |

**Весь этот путь ты пройдёшь в курсе** — шаг за шагом.

## Глава 4 — Файловая система Linux: всё есть файл

<div align="center">
<img src="https://raw.githubusercontent.com/OlegKarenkikh/devops-for-kids/main/images/intro_filesystem.png" alt="Файловая система Linux" width="90%"/>
<br/><em>В Linux всё — файл: настройки, программы, устройства, процессы</em>
</div>

### Метафора: библиотека

Файловая система Linux — это библиотека с чёткой системой полок:

```
/                        ← Корень (root) — самая верхняя папка
├── home/                ← Домашние папки пользователей
│   └── user/            ← Твоя папка (как стол в библиотеке)
│       ├── projects/    ← Твои проекты
│       └── .ssh/        ← SSH ключи (точка = скрытая папка)
├── etc/                 ← Настройки системы (конфиги программ)
│   ├── nginx/           ← Настройки веб-сервера Nginx
│   └── hosts            ← Таблица DNS имён
├── var/                 ← Переменные данные (логи, базы)
│   └── log/             ← Журналы событий
├── usr/                 ← Программы пользователей
│   └── bin/             ← Исполняемые файлы (python3, git, curl...)
└── tmp/                 ← Временные файлы (удаляются при перезагрузке)
```

### Важные правила Linux

| Правило | Пример | Почему важно |
|---------|--------|-------------|
| **Регистр важен** | `README.md ≠ readme.md` | Два разных файла! |
| **Точка = скрытый файл** | `.env`, `.gitignore` | `ls` не покажет, нужен `ls -la` |
| **/ = корень** | `/home/user` | Всегда начинай путь от корня |
| **~ = твоя папка** | `cd ~` = `cd /home/user` | Быстрый возврат домой |
| **. = текущая папка** | `./app.py` | Запустить файл тут |
| **.. = папка выше** | `cd ..` | Подняться на уровень выше |

## Глава 5 — Как читать и писать конфиги: YAML, ENV, JSON

<div align="center">
<img src="https://raw.githubusercontent.com/OlegKarenkikh/devops-for-kids/main/images/intro_yaml_rules.png" alt="YAML правила" width="90%"/>
<br/><em>YAML — самый читаемый формат конфигов. Главное правило: отступы через пробелы!</em>
</div>

### Форматы конфигурационных файлов

В DevOps ты встретишь три главных формата. Не бойся — они все простые:

#### YAML — для Docker Compose и Kubernetes

```yaml
# Символ # — это комментарий, объяснение

# Ключ: значение
имя: Флот Python
версия: "3.9"          # Строки можно в кавычках

# Список (начинается с -)
зависимости:
  - flask
  - requests
  - psycopg2

# Вложенный объект (отступ = 2 пробела)
база_данных:
  хост: localhost      # 2 пробела!
  порт: 5432           # 2 пробела!
  имя: mydb            # 2 пробела!
```

> ⚠️ **Главная ошибка YAML**: вместо 2 пробелов поставить Tab. Используй только пробелы!

#### ENV — для секретов и настроек

```bash
# Файл .env — переменные окружения
# Формат: ИМЯ=значение (без пробелов вокруг =)

APP_NAME=МойСервис
APP_PORT=8080
DB_PASSWORD=СекретныйПароль
DEBUG=false
```

#### JSON — для API ответов и настроек

```json
{
  "имя": "МойСервис",
  "порт": 8080,
  "зависимости": ["flask", "requests"],
  "база": {
    "хост": "localhost",
    "порт": 5432
  }
}
```

## Глава 6 — Порты: номера квартир на сервере

<div align="center">
<img src="https://raw.githubusercontent.com/OlegKarenkikh/devops-for-kids/main/images/intro_ports.png" alt="Порты" width="90%"/>
<br/><em>Сервер — это дом. IP-адрес — адрес дома. Порт — номер квартиры</em>
</div>

### Метафора: почтовые ящики

IP-адрес `192.168.1.100` — это адрес дома. Порт `:8080` — номер квартиры. Письмо (запрос) приходит на адрес дома и идёт в конкретную квартиру.

```
http://localhost:8080/items
  │        │       │    │
  │        │       │    └── Путь — конкретная страница/ресурс
  │        │       └────── Порт — какой сервис слушает
  │        └────────────── Хост — адрес сервера (localhost = твой компьютер)
  └─────────────────────── Протокол — как общаться
```

### Стандартные порты — выучи наизусть

| Порт | Сервис | Запомни как |
|------|--------|------------|
| **80** | HTTP (сайт) | Стандартный — браузер подставляет сам |
| **443** | HTTPS (защищённый) | Замочек в браузере |
| **22** | SSH | Удалённый терминал |
| **5432** | PostgreSQL | Наша база данных |
| **6379** | Redis | Кэш |
| **8080** | Наш API | Договорились так в курсе |
| **9090** | Prometheus | Мониторинг |
| **3000** | Grafana | Графики |

## Глава 7 — Переменные окружения: секреты приложения

<div align="center">
<img src="https://raw.githubusercontent.com/OlegKarenkikh/devops-for-kids/main/images/intro_env_vars.png" alt="Переменные окружения" width="90%"/>
<br/><em>Код — в GitHub (публично). Секреты — в .env (приватно). Никогда не смешивай!</em>
</div>

### Почему нельзя хранить пароли в коде?

В 2022 году разработчик случайно закоммитил AWS-ключ в публичный GitHub. Через **4 минуты** боты нашли ключ и потратили $50 000 на облачные ресурсы. Компания получила счёт.

**Правило:** секреты — пароли, API-ключи, токены — **никогда** не попадают в код.

```python
# ❌ Опасно — пароль виден всем в GitHub
db_password = "МойСекретныйПароль123"

# ✅ Правильно — пароль читается из окружения
import os
db_password = os.environ.get("DB_PASSWORD", "default_for_dev")
```

```bash
# Файл .env (НЕ коммитится в Git)
DB_PASSWORD=МойСекретныйПароль123
APP_SECRET=jwt_secret_key_here

# Файл .gitignore (коммитится — указывает что игнорировать)
.env
*.pyc
__pycache__/
```

### Цепочка передачи секретов

```
.env файл (локально)
    ↓ docker compose (читает .env)
Контейнер получает переменные
    ↓ os.environ.get() в Python
Приложение использует значение
    ↓ (в продакшене)
Kubernetes Secret → Pod → контейнер → приложение
```

## Глава 9 — Психология обучения: как не бросить

### 🧠 Три правила которые работают

**1. Ошибки — это нормально.** Каждый программист в мире ошибается каждый день. Сообщение об ошибке — это не провал, это подсказка. Читай его полностью, особенно последнюю строку.

**2. Практика важнее теории.** Прочитать про `git commit` и сделать `git commit` — разные вещи. Каждую команду набирай руками, не копируй. Мышечная память важна.

**3. Маленькие победы.** Каждый запущенный контейнер, каждый сделанный коммит — это победа. Курс разбит на маленькие шаги специально.

### 🔄 Алгоритм при ошибке

```
Получил ошибку
     │
     ▼
Прочитай последнюю строку ошибки — там суть
     │
     ▼
Погугли ТОЧНЫЙ текст ошибки (в кавычках)
     │
     ▼
Попробуй решение → сработало? → Идёшь дальше 🎉
     │ нет
     ▼
Проверь: опечатки? отступы? права доступа?
     │
     ▼
Спроси в чате курса — никто не застрял в одиночку
```
